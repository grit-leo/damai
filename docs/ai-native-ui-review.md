# 麦总 AI 审核助理 UI Review

## 目标

把新品审核插件从“右侧表单工具”升级为“AI Native 审核助手”：用户打开插件后，应能像交代任务一样让麦总读单、补证据、算风险、出意见。

## 当前体验结论

当前版本已经完成第一阶段转型：

- 首屏是 Agent 工作台，而不是字段表单。
- 打开助手后，原审批页会整体左移，助手以停靠工作区出现，不再像浮层压在页面上。
- 麦总会展示“拾取新品信息”的动态过程，让用户感知系统正在理解页面。
- 麦总会展示当前审核状态、关键价格指标和三步任务计划。
- 用户可以输入自然语言指令，例如“帮我完整审核这单”。
- 系统会自动串联查询官旗价、采集详情图、计算风险并生成审核意见。
- 分析完成后会在首屏展示结论快照，用户不用向下翻找最终判断。
- 审批字段被降级为“数据明细”，技术服务地址改为隐藏配置，避免用户看到代码感字段。

## 本轮发现的问题

1. 旧版首屏仍以表单为主，用户第一眼看到的是字段编辑，而不是 AI 助手在工作。
2. “AI 助手”只有视觉包装，没有自然语言任务入口，用户仍要理解按钮顺序。
3. 采图、查价、分析之间缺少连贯的 Agent 状态反馈。
4. 移动窄屏下顶部副标题容易被挤压，影响质感。
5. 自动化验收只覆盖按钮路径，未覆盖最重要的自然语言审核路径。
6. 侧栏覆盖在原页面上，用户会感觉是浮层干扰，而不是工作区扩展。
7. 本地服务地址、示例数值等技术信息暴露在界面里，降低真实产品感。

## 已完成改进

- 增加麦总 Agent 状态条：展示当前阶段和下一步。
- 增加三步计划：读单完成、补证据、出意见。
- 增加自然语言指令框和快捷指令。
- 实现“帮我完整审核这单”的任务编排。
- 增加页面左移停靠布局：打开助手时审批页让出右侧工作区。
- 增加新品信息拾取动效：商品名、商品编码、采购价、京东价逐项展示已拾取状态。
- 增加证据槽，把官旗价、全网低价、BOM 放到首屏核心位置。
- 增加证据状态：官旗价、详情图、BOM。
- 增加首屏结论快照：风险、建议采购价、风险分和一键复制。
- 将审批单字段移动到数据明细，将技术服务地址改为隐藏配置。
- 修复移动端顶部副标题挤压问题。
- 补充自然语言路径自动化验收。
- 补充布局左移、拾取状态、技术字段不外露的自动化验收。

## 高级产品与交互 Review

1. 信息架构：现在首屏优先回答“我是谁、我在干什么、我读到了什么、下一步要补什么”，比字段表单更符合 AI Native 助手心智。
2. 空间关系：停靠式侧栏比覆盖浮层更好。它让用户相信原审批页仍是主工作对象，助手只是扩展工作区，不会挡住原页面证据。
3. 智能感：拾取动效比静态“已识别”更强。用户打开插件后能看到商品名、编码、采购价、京东价被逐项捕获，形成“助手正在读页面”的即时反馈。
4. 信任感：隐藏本地服务地址和技术配置是必要的。采销用户不应该看到接口、localhost、端口号这类工程信息；他们只需要看到证据是否已补齐。
5. 决策效率：首屏结论快照是正确方向。审核场景里用户最关心风险、建议采购价、能否复制意见，不应该先被完整表格消耗注意力。
6. 剩余风险：当前自然语言能力仍是规则匹配，尚不是完整 LLM Agent。生产版需要真实意图理解、追问能力、证据置信度解释和审批结果回写。

## 后续专业建议

1. 接入真实 LLM/Agent 后端，让麦总能理解开放式问题，而不是只匹配固定关键词。
2. 将竞对价、相似 SKU、历史费用率和 BOM 成本接入内部接口，减少人工补证据。
3. 增加证据置信度和缺失证据提醒，避免用户误把低置信结论当最终审批依据。
4. 输出拆成三类：Boss 审批意见、品牌谈判话术、证据摘要。
5. 增加任务时间线和可追溯证据包，支持审批复盘。
6. 允许用户继续追问，例如“为什么建议驳回”“如果采购价降到 22 元还能过吗”。

## 产品判断

当前版本已经具备 AI Native 的前端体验和任务编排雏形；生产级版本还需要真实 LLM 推理、内部数据源和权限体系支撑。
